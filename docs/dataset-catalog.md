# LLM 评估数据集分类列表

## 一、通用能力与知识基准（Foundation Benchmarks）

| # | 名称 | 描述 | 含金量 |
|---|------|------|--------|
| 1 | **MMLU-Pro** | MMLU 的高难度升级版，覆盖 57 个学科的多选题，用于评估通用知识与多学科理解。 | ⭐⭐⭐⭐ |
| 2 | **GPQA Diamond** | 博士级科学推理基准，包含物理、化学、生物领域的专家出题，当前最具含金量的学术基准之一。 | ⭐⭐⭐⭐⭐ |
| 3 | **HLE（Humanity's Last Exam）** | 全人类知识极限测试，3000 道极难问题，用于探查 LLM 的能力边界。 | ⭐⭐⭐⭐⭐ |
| 4 | **AIME 2025** | 美国数学邀请赛真题，评估高难度数学竞赛推理能力。 | ⭐⭐⭐⭐⭐ |
| 5 | **SWE-bench Verified** | 基于真实 GitHub Issue，衡量模型解决软件工程问题的能力。 | ⭐⭐⭐⭐ |
| 6 | **LiveBench** | 定期更新题目的动态基准，防止数据污染，反映真实泛化能力。 | ⭐⭐⭐⭐ |
| 7 | **FACTS** | 评估模型在闭卷、开卷、长篇生成等场景下的事实准确性。 | ⭐⭐⭐ |
| 8 | **SciCUEval** | 科学文献语境理解基准，涉及知识图谱，评估信息的识别、整合与推理。 | ⭐⭐⭐ |
| 9 | **CORE** | 包含 22.5 万选择题，覆盖 74 个学科，评估深层语义推理与本体关系。 | ⭐⭐⭐⭐ |
| 10 | **BIG-bench** | 大规模协作基准，包含超 200 个多样化任务，考验综合能力。 | ⭐⭐⭐⭐ |

## 二、RAG（检索增强生成）专用基准

| # | 名称 | 描述 |
|---|------|------|
| 1 | **MIRAGE** | 可分别评估检索和生成两个环节的独立表现，包含 7560 个问答对。 |
| 2 | **CReSt** | 模拟真实 RAG 场景，评估复杂推理、答案拒答、精确引用和文档结构理解。 |
| 3 | **RAGuard** | 首个系统评估 RAG 系统对误导性检索结果鲁棒性的基准。 |
| 4 | **RARE** | 通过带噪声的查询和文档进行压力测试，考察 RAG 在动态环境下的稳定性。 |
| 5 | **ChronoQA** | 专门评估 RAG 系统中的时间推理能力，覆盖绝对、相对和聚合时间问题。 |

## 三、GraphRAG（基于知识图谱的 RAG）评估基准

| # | 名称 | 描述 |
|---|------|------|
| 1 | **GraphRAG-Bench** | 领域内最知名的专用基准，评估大学水平的多跳推理，覆盖图构建、检索到生成的端到端过程。 |
| 2 | **M³GQA** | 聚焦多实体、多约束的复杂查询，推动 GraphRAG 方法的能力边界。 |
| 3 | **RiTeK** | 垂直领域先驱，评估医学文本知识图谱上的复杂路径推理能力。 |
| 4 | **MultiHal (KG-RAG)** | 首个多语言场景下基于知识图谱的幻觉评估数据集，验证 KG 集成对事实一致性的提升。 |
| 5 | **Unbiased Eval Framework** | 一套关注评估公平性的框架，从可追溯性、来源对齐和可解释性三个维度消除评估偏见。 |

## 注水检测优先级建议

对于检测 LLM 是否注水（用弱模型冒充强模型），优先使用以下数据集：

1. **GPQA Diamond** — 博士级难度，弱模型无法蒙混，区分度最高
2. **MMLU-Pro** — 专业学科深度，简单模型明显掉分
3. **AIME / MATH** — 数学推理，弱模型几乎无法完成
4. **BBH（BIG-bench Hard）** — 多步推理，对模型能力敏感
5. **LiveBench** — 防污染，无法靠记忆答题

> 当前项目 starter 集中的 gpqa/mmlu-pro 题目为简化版常识题，不具备真实区分度。
> 见 `benchmarks/starter/` 目录，建议逐步替换为真实数据集样本。
